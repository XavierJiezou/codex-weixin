# 微信端模型与推理强度切换设计

## 目标

允许用户直接在微信私聊中查看并切换 Codex 模型与推理强度，同时保持多微信账号、多联系人和多会话之间互不影响。

## 方案

- 新增 `/model` 和 `/effort` 命令；不带参数时展示当前值和编号选项，支持编号、英文 ID，以及 `default` 恢复继承设置。
- 设置保存在当前受管会话中。新会话不复制旧会话的覆盖值，而是继承 Web 全局配置或 Codex 自身配置。
- 微信模型列表复用 Web 设置页的模型目录，包括提供方补充的 GPT-5.6 Sol、Terra 和 Luna。
- app-server 与 `codex exec` 回退都接收会话的模型和推理强度。Web 继续同一会话时也使用这份会话覆盖值。
- `/status` 按“会话覆盖 > Web 全局配置 > Codex 运行时配置”的顺序显示实际生效值。

## 兼容性与失败处理

- 旧状态文件没有 `model`、`effort` 字段时继续按原逻辑继承全局设置。
- 如果暂时无法读取模型目录，仍允许输入完整模型 ID；推理强度则提供 Codex 常见等级作为回退。
- 切换模型时，如果当前实际推理强度不被新模型支持，则自动改为该模型默认（或第一个可用）强度并提示用户，避免下一次运行提交无效组合。
