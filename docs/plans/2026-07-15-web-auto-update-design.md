# Web 自动更新设计

## 目标

codex-weixin 在 Web 管理页面启动后检查 npm Registry 的稳定最新版。发现高于当前版本的新版本时，主动展示更新弹窗，由用户选择稍后处理或立即更新。立即更新成功后自动重启本地服务，Web 等待服务恢复并刷新页面。

## 服务端

新增独立更新管理器，负责严格的稳定版语义版本比较、短时结果缓存和全局 npm 安装。后台并行探测 npm 官方源与 npmmirror 国内镜像，自动采用最快返回合法稳定版本的源；如果最快结果没有新版本，还会等待另一个源确认，避免镜像同步延迟造成漏报。用户无需配置或选择镜像，安装阶段固定沿用本次检查选中的内置源。

浏览器不能提交包名、版本、命令或 Registry 地址；安装目标只能是服务端刚刚校验过的 `codex-weixin@<latest>`。更新接口继续受现有本机 Host、Origin 和随机请求令牌保护，同一时间只允许一个安装任务。两个 Registry 都不可用时返回无更新状态，不影响账号、会话或首页加载；安装失败时保留当前进程并给出可操作错误。

安装成功后，主进程启动一个与自身分离的重启助手，再优雅停止微信监控、HTTP 服务并释放进程锁。重启助手等待旧进程退出后，通过 Node.js 直接运行更新后的服务入口，继承端口和状态目录，并禁止重复打开浏览器。该方式避免依赖 Windows 的 `.cmd` 直接启动行为。

## Web 交互

Web 在主界面完成初始化后异步检查更新，不阻塞正常使用。弹窗明确展示当前版本和目标版本，提供“稍后”和“立即更新”。选择稍后时，仅在浏览器本地记录已忽略版本；更高版本仍会再次提示。更新期间按钮锁定并展示安装、重启和恢复状态。服务重启后请求令牌会变化，Web 通过无权限的启动接口确认目标版本已运行，再自动刷新。

## 验证

测试覆盖版本比较、双 Registry 自动选择、镜像同步延迟、Registry 响应校验、缓存、并发安装保护、固定安装目标、更新接口权限以及回调触发。正式构建后检查桌面和窄屏弹窗布局，并验证更新失败不会停止现有服务。
