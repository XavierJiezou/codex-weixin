# 微信账号授权状态设计

## 目标

账号管理页不再把内部 sender 映射成“微信联系人 1”展示。该名称不能帮助用户识别真实身份，还容易被误解为已经获取了微信联系人资料。账号卡片只回答用户最关心的问题：这个账号现在能否从微信控制 Codex。

## 状态

- `已授权`：账号至少存在一个允许的微信 sender，可正常控制 Codex，并提供“撤销授权”。
- `待授权`：尚未授权，但已经收到新的微信访问请求，提供“允许访问”。
- `未授权`：没有已授权 sender，也没有待处理请求，提示用户先从微信发送一条消息。

账号已授权后如果又出现新的 sender 请求，继续保留单独的待授权提醒，不暴露内部 ID，也不影响已有授权。

## 操作

“撤销授权”按账号撤销当前全部允许的 sender，并在执行前明确确认。内部 `@im.wechat` 标识仍只用于本机 API 路由，不显示给普通用户。允许新请求仍使用现有本机令牌保护的 sender 授权 API。

## 验证

检查已授权账号只显示授权状态，不出现“微信联系人 1”或内部 ID；在窄屏下状态说明和操作按钮保持完整。现有授权 API 测试继续覆盖允许和撤销行为，构建后通过真实管理页面确认布局。
