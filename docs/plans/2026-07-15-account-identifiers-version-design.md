# Web 账号标识与版本号展示设计

## 信息层级

- codex-weixin 版本号属于全局产品信息，显示在页面顶部品牌名称旁，以紧凑徽标呈现。
- Bot ID 和 User ID 属于排障信息，不替代账号备注。每张账号卡片增加原生可展开区域，默认收起以保持列表易读。

## 数据来源

- 服务端从随 npm 包发布的 `package.json` 读取版本，并通过 `/api/bootstrap` 返回。
- Bot ID 使用扫码返回并随凭据刷新的 `botId`，User ID 使用稳定的 `userId`；本地 `accountId` 继续作为保留会话数据的稳定主键。旧账号缺少 `botId` 时回退显示 `accountId`。微信凭据 token 仍不返回浏览器。

## 交互与响应式

- 展开“账号 ID”后同时显示完整 Bot ID 与 User ID，等宽字体排版并允许直接选择复制。
- 使用原生 `details/summary` 保留键盘操作和可访问性；展开箭头明确反馈状态。
- 账号列表后台刷新时保留每张卡片的展开状态，避免用户查看过程中被自动收起。
- 桌面端两列显示两个 ID，窄屏改为单列，长 ID 自动换行而不截断。

## 验证

- API 测试确认版本字段存在且 token 继续脱敏。
- 桌面与移动视口检查版本徽标、展开状态、完整 ID 和卡片对齐。

## 会话模型设置补充

- 会话聊天标题下方增加模型和推理强度下拉框，选择后自动保存到当前受管会话。
- 下拉框支持继承全局设置，并按当前模型动态展示推理强度；不兼容时自动调整到模型默认强度。
- Web 控件与微信 `/model`、`/effort` 使用同一份会话覆盖值，因此两端保持一致。
